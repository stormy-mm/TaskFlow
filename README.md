TaskFlow — это консольное приложение для управления задачами (to-do), построенное по слоям `CLI -> Application -> Domain -> Repository`.

Приложение позволяет:
- создавать задачи;
- запускать/завершать/отменять задачи;
- редактировать поля задачи;
- просматривать одну задачу или весь список;
- удалять задачи и очищать список;
- сохранять задачи между запусками в `tasks.json`.

## Возможности

- **Статусы задач**: `NEW`, `IN_PROGRESS`, `DONE`, `CANCELLED`, `OVERDUE`.
- **Обычные и дедлайновые задачи**:
  - без дедлайна — обычная задача;
  - с дедлайном — проверяется срок при смене статуса.
- **Валидация переходов статусов**:
  - нельзя завершить `NEW`/`CANCELLED`;
  - нельзя запустить `DONE`/`CANCELLED`;
  - нельзя отменить `DONE`/`CANCELLED`.
- **Персистентность**: JSON-репозиторий автоматически загружает/сохраняет задачи.

## Как работать с приложением

### Основные команды

- `add` — добавить задачу (интерактивный ввод полей).
- `start <id>` — перевести задачу в `IN_PROGRESS`.
- `complete <id>` — перевести задачу в `DONE`.
- `cancel <id>` — перевести задачу в `CANCELLED`.
- `edit <id>` — редактировать задачу (вложенная команда).
- `show <id>` — показать задачу по id.
- `list` — показать все задачи.
- `delete <id>` — удалить задачу.
- `clear` — удалить все задачи.
- `exit` — выйти из программы.

### Формат дедлайна

При вводе дедлайна используйте формат:

`ДД ММ ГГГГ ЧЧ ММ СС`

Пример:

`12 1 2027 8 0 0`

Если дедлайн не нужен — нажмите `Enter` на шаге ввода дедлайна.

### Редактирование задачи (`edit <id>`)

После ввода команды `edit <id>` приложение попросит указать атрибут и новое значение:

- `id <new_id>`
- `title <new_title>`
- `description <new_description>`
- `deadline <ДД ММ ГГГГ ЧЧ ММ СС>`

## Хранение данных

- По умолчанию задачи сохраняются в файл `tasks.json` в директории запуска.
- Файл создаётся автоматически при первом изменении списка задач.
